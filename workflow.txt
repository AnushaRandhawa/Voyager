WORKFLOW 1: BY ANUSHA

Name: Collaborative Expense Splitting
Actor: Trip Participant / Payer
Pre-condition: User is logged into the app and is a member of an active "Trip Group."

Steps:
1. User navigates to the Expenses tab and clicks the "+" (Add Expense) button.
2. User enters the total amount spent (e.g., 5000 PKR) and selects a category (e.g., Transport).
3. User selects the specific group members involved in this specific expense.
4. User chooses the Split Equally calculation method.
5. User uploads a photo of the receipt (optional) and clicks Confirm.
6. The system calculates the individual shares and updates the "Who Owes Whom" dashboard for every member in the group.

Post-condition: The central ledger is updated, and all involved members receive a notification of the new pending debt.

WORKFLOW 2: BY TAHERA

Name: Shared Trip Essentials Checklist
Actor: Trip Organizer / Group Member
Pre-condition: A trip destination has been created in the app.

Steps:
1. User opens the Checklist section and selects the "Create New List" option.
2. User enters a title for the checklist (e.g., "Camping Gear") and adds items such as "Tent" and "Sleeping Bag."
3. Another group member views the checklist in real-time and clicks the "Assign to Me" option for a specific item (e.g., "Tent").
4. As members pack items, they click the checkbox next to each item to mark it as completed.
5. The system updates and displays a progress bar (e.g., "75% Packed") visible to all group members.
6. The system detects any "Unassigned" items 24 hours before the trip and sends a reminder notification to the group.

Post-condition: The checklist is synchronized in real-time, and the group has a clear view of trip readiness, minimizing the risk of forgotten items.

WORKFLOW 3: BY HIBA

Name: Daily Itinerary & Schedule Management
Actor: Trip Lead / Group Member
Pre-condition: The trip dates have been set within the application.

Steps:

1. User navigates to the Timeline view and selects a specific day of the trip.
2. User clicks Add Event to create a time-blocked slot (e.g., "08:00 AM - Breakfast at Hotel").
3. User adds a location (e.g., Departure for Neelum Valley").
4. The system saves the event and syncs it to every group member's personal in-app calendar.
5. The system triggers a push notification to all members 15 minutes before the next scheduled activity.

Post-condition: All travelers are kept on the same page regarding the daily plan, reducing delays and communication gaps.
